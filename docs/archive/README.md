# Archived design records

This directory preserves the documents used to design and sequence the initial
`flower-ai-harness` implementation.

They are useful for understanding why the project has its current boundaries,
but they are not maintained as exact descriptions of the code. Several contain
future-tense sketches, phase checklists, or assumptions that were later
superseded.

Use the maintained documentation instead:

- [`../PROJECT_OVERVIEW.md`](../PROJECT_OVERVIEW.md) for the project concept and
  boundary.
- [`../ARCHITECTURE.md`](../ARCHITECTURE.md) for the current execution and
  dependency architecture.
- [`../MODULES.md`](../MODULES.md) for the current module catalog.
- [`../IMPLEMENTATION_STATUS.md`](../IMPLEMENTATION_STATUS.md) for completed,
  active, and deferred work.
- [`../plans/`](../plans/) for active implementation plans.

Archived files:

- `ARCHITECTURE.md`: initial v0 architecture.
- `API_DESIGN.md`: initial public API sketches.
- `IMPLEMENTATION_PLAN.md`: initial implementation strategy and risk analysis.
- `PHASED_IMPLEMENTATION.md`: initial phase-by-phase worksheet.
- `STRATEGY.md`: early positioning and roadmap notes.
- `OPEN_SOURCE_STRATEGY.md`: early open-source and commercialization notes.
- `AGENT_CLI_PROVIDER_PLAN.md`: completed plan for the external agent
  subprocess provider.
